# Codemod commands

The codemod is mechanical but spans more file types than you'd expect. This doc enumerates every place the version string lives.

## The version string

The package name `@gusto/embedded-api-v-<DATE>` appears in:

1. **Import paths** — every `import ... from '@gusto/embedded-api-v-<OLD>/...'`
2. **Hardcoded `queryKey` literals** — `queryKey: ['@gusto/embedded-api-v-<OLD>', ...]`
3. **JSDoc references** in `createSdkQueryClient.ts:4` and `ApiProvider.tsx:35` docstring
4. **CLAUDE.md / AGENTS.md / src/CLAUDE.md** — instruction file references
5. **`docs/api/*.md` / `docs/reference/*.md`** — partner-facing docs
6. **`build/deriveEndpointInventory.ts`** — derive script
7. **`e2e/utils/validationErrorCollector.test.ts`** — e2e helper test
8. **`package.json`** — dependency name

The single sed sweep below catches 1, 2, 3, 6, 7. Steps 4, 5, 8 need separate handling.

## The main sweep

```bash
OLD="2025-11-15"  # ← set per upgrade
NEW="2026-02-01"

find src sdk-app -type f \( -name "*.ts" -o -name "*.tsx" \) -exec \
  sed -i '' "s|@gusto/embedded-api-v-${OLD}|@gusto/embedded-api-v-${NEW}|g" {} +
```

This handles ~360 files. The same regex catches import paths AND `queryKey` array literals because they share the version string.

### Migrating FROM the bare/unversioned package

When `OLD` is the **unversioned** `@gusto/embedded-api` (as in the original v0.13.0 → v2025-11-15 migration), the standard sed above is **dangerous**: the source string is a prefix of every dated version (`@gusto/embedded-api-v-2025-11-15` contains `@gusto/embedded-api`), so a naive global replace would corrupt any already-dated reference it finds.

Use a **boundary-anchored** regex when migrating from the bare package. The bare package is always followed by `/`, `'`, `"`, or end-of-line in real code — never by `-v-`. Anchor on those follow characters:

```bash
# When OLD is the bare package
find src sdk-app -type f \( -name "*.ts" -o -name "*.tsx" \) -exec \
  sed -i '' -E "s|@gusto/embedded-api([/'\"\\])|@gusto/embedded-api-v-${NEW}\\1|g" {} +
```

The `([/'\"])` capture group preserves whatever followed the bare name. **Verify with grep before committing:**

```bash
# Should return zero hits after the sweep
grep -rln '@gusto/embedded-api[^-]' src sdk-app
```

This step does not apply when migrating between two already-dated versions — those don't share a prefix.

## Explicit docs + build sweeps

```bash
sed -i '' "s|@gusto/embedded-api-v-${OLD}|@gusto/embedded-api-v-${NEW}|g" \
  AGENTS.md \
  CLAUDE.md \
  src/CLAUDE.md \
  docs/api/*.md \
  docs/reference/*.md \
  build/deriveEndpointInventory.ts \
  e2e/utils/validationErrorCollector.test.ts
```

**Do NOT touch:**

- `CHANGELOG.md` — historical; previous version mentions are correct as-is.
- `.reports/*` — auto-generated by `npm run derive`; will regenerate.
- `.claude/worktrees/` — sibling worktrees; not part of this checkout.

## CRITICAL — the bare-date `X-Gusto-API-Version` sweep

This is the single highest-risk silent-failure site in any upgrade. The SDK sends the API version as an HTTP header on every request, sourced from a **bare date string** that is NOT part of any package name:

```ts
// src/contexts/ApiProvider/apiVersionHook.ts:3
const CURRENT_API_VERSION = '2025-11-15' // ← MUST flip per upgrade
```

The package-name sed sweep above does **not** match this string because it's a date literal, not a package path. If you skip this step, the upgrade is a **runtime no-op**: types claim the new version, but `X-Gusto-API-Version` still sends the old version, and the server answers with the old schemas regardless. Tests pass, typecheck passes, e2e may pass — and partners get the wrong API contract.

### The mandatory sites

```bash
OLD="2025-11-15"
NEW="2026-02-01"

# Source-of-truth constant
sed -i '' "s|'${OLD}'|'${NEW}'|g" src/contexts/ApiProvider/apiVersionHook.ts

# Test assertions (3 sites in apiVersionHook.test.ts, 1 site in ApiProvider.test.tsx as of v2026-02-01)
sed -i '' "s|'${OLD}'|'${NEW}'|g" \
  src/contexts/ApiProvider/apiVersionHook.test.ts \
  src/contexts/ApiProvider/ApiProvider.test.tsx
```

### Verification

```bash
# Should return only Storybook story-date fixtures, never SDK runtime code
grep -rn "'${OLD}'\|\"${OLD}\"" src/ sdk-app/ e2e/ | grep -v node_modules
```

The only acceptable hits are unrelated Storybook story fixtures using the date as a calendar value (e.g. `endDate: '2025-11-15'` in payroll story files). Any hit in `src/contexts/ApiProvider/`, `src/hooks/`, or anywhere outside `*.stories.*` is a real miss and MUST be flipped before the codemod commit.

## `package.json` edit

Use `Edit`, not sed (sed is dangerous on JSON because it can break the trailing comma rules). Look up the actual published version of the new package first:

```bash
npm view @gusto/embedded-api-v-${NEW} versions --json
```

Then replace the line. Typical shape:

```diff
-    "@gusto/embedded-api-v-2025-11-15": "^0.0.2",
+    "@gusto/embedded-api-v-2026-02-01": "^0.0.1",
```

After editing `package.json`, run `npm install` so `package-lock.json` regenerates. Stage both files together.

## Verify the sweep is complete

```bash
grep -rln "@gusto/embedded-api-v-${OLD}" . \
  --exclude-dir=node_modules \
  --exclude-dir=.git \
  --exclude-dir=.claude \
  --exclude-dir=.reports \
  --exclude=package-lock.json \
  --exclude=CHANGELOG.md
```

Expected output: **empty**. If anything remains, address each hit explicitly before committing.

## Cache-namespace hot-spots (historical)

These are the non-obvious places hardcoded `'@gusto/embedded-api-v-<OLD>'` string literals showed up in the v2025-11-15 → v2026-02-01 sweep. The main sed catches them, but knowing where they live helps verify:

- `sdk-app/src/design/prototypes/contractor-management/AddContractor/index.tsx`

The TimeOff and Company/Locations sites are the highest-risk — they're production code and a missed flip means mutations under the new namespace stop invalidating those queries.

## Stage safely

After the sweep, use `git add -u` (modified-files-only), **never** `git add -A`. The latter picks up `.claude/worktrees/` as submodules and bloats the commit with 30+ embedded git directories. If you accidentally do `add -A`, recover with:

```bash
git reset HEAD .claude/worktrees/
```
